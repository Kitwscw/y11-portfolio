project 1: Low poly character

Overview:


this project is to create a low poly character for use in future projects eg: like video games & stuff i want to make like a knight guy modelled after a medieval knight from... medieval times

Final outcome

its not finished yet ok

end users

my end users are game developers making a game set in medieval times these kinds of people tend to want characters that have already been rigged with say, a running animation or a jumping one or one for attacking maybe.

requirements

The character must have:

  • head
    • helmet
    • eye holes
  • torso
    • armour
    • a weapon
  • arms & legs
    • armour on them
    • fingers
    • gloves
    • boots
    Rigging & animations
    • basic rigging
    • running animation
    • jumping animation
    • idle animation

Implications

  • Aesthetics
    • aesthetics is important because a good looking main character in a video game is what draws people to the game & is usually what people think of first when thinking about the game
  • Functionality
    • Functionality is important because if the character model doesn't work then there is no reason to use it because why would you use a nonfunctional tool?
  • Usability
    • Usability is important because if you can't use it then there isn't much reason to have it because well, its useless
